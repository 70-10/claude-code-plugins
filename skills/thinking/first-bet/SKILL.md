---
name: first-bet
description: Help the user identify the first hypothesis worth testing when the right answer is unclear.
---

Interview the user to build a shared understanding and identify the first hypothesis worth testing.

Ask questions one at a time, waiting for feedback before continuing.

Focus on the hypothesis whose outcome would most change what happens next. Make it specific enough to be supported or rejected by the test.

If a question can be answered by exploring the codebase, explore the codebase instead.

Stop when the user agrees on the hypothesis, how to test it, and the conditions for supporting or rejecting it. If there is not enough information to form a hypothesis, state what is missing and stop.

Do not run the test or enact the plan.
